Overview

This section presents the practical machine learning activities completed throughout the module. These activities demonstrate my progression from foundational data exploration and regression to clustering, neural networks, deep learning, and model evaluation. More importantly, they show how my understanding developed beyond implementation into interpretation, model limitations, and real-world application.

Unit 2 & 3: Exploratory Data Analysis and Regression

I conducted exploratory data analysis (EDA) on the Auto MPG dataset, beginning with data cleaning and handling missing values. Statistical measures such as skewness and kurtosis were analysed to understand data distribution, which is critical before applying any machine learning model.

A key outcome of this analysis was identifying a strong negative correlation between vehicle weight and fuel efficiency (MPG). This relationship was visualised using scatter plots and correlation heatmaps, helping to translate statistical findings into interpretable insights. The uploaded evidence shows the missing-value check, skewness and kurtosis output, a correlation heatmap, a scatter plot, covariance and Pearson correlation, and simple linear regression output.

A simple linear regression model was then implemented to quantify this relationship. While the model captured the overall trend, this activity highlighted an important limitation: relying on a single predictor reduces model accuracy. This reinforced the importance of feature selection and the need for more complex models in real-world scenarios. The notebook evidence explicitly reports a Pearson correlation of about -0.83 between weight and MPG, supporting this interpretation.

Unit 4: Linear Regression with Scikit-Learn

In Unit 4, I extended my understanding of regression by working with larger-scale data on population and GDP. This activity moved beyond simple feature-level relationships and introduced the challenge of modelling real-world macro-level data where multiple factors influence the outcome.

The analysis involved merging datasets, calculating mean population and mean GDP, measuring correlation, and then fitting a linear regression model. The uploaded output shows a Pearson correlation of approximately 0.72, suggesting a moderately strong positive relationship, but it also shows that the regression model explained only part of the variation, with an R² of about 0.52 and relatively large error values. This was important because it demonstrated that visible correlation does not automatically imply that a single-variable model is sufficient for accurate prediction.

This activity strengthened my ability to interpret model output critically rather than assuming that regression itself guarantees strong predictive power. It also reinforced a wider lesson from the module: performance depends not only on the algorithm, but also on the completeness and suitability of the data.

View Unit 4 Regression Evidence

Unit 6: Clustering and Unsupervised Learning

This activity introduced unsupervised learning through K-Means clustering using the Iris dataset. Unlike regression, clustering does not rely on labelled outputs. Instead, it identifies patterns in the data based on similarity, which makes it useful for exploratory analysis and segmentation.

Before applying K-Means, the dataset was standardised so that all features contributed equally to the model. The elbow method was then used to determine an appropriate number of clusters, and the resulting clustering was evaluated using the silhouette score. The evidence shows the elbow plot, the cluster visualisation, and the silhouette score output, making it possible to see both the modelling process and its evaluation.

This activity helped me understand that clustering is not simply about running an algorithm. It also requires interpretation, judgement, and awareness of how preprocessing decisions affect the final result. It strengthened my understanding of how unsupervised learning can be applied in practical contexts such as segmentation, pattern discovery, and anomaly detection.

View Unit 6 Clustering Evidence

Unit 7: Neural Networks – Perceptron Activities

Unit 7 marked an important transition from classical machine learning techniques toward neural networks. Through the perceptron activities, I explored how weighted inputs, activation functions, and iterative updates allow a model to learn from data.

The first activity implemented a simple perceptron using NumPy and showed how changing the weights changed the final output. The second activity used a perceptron to model the AND operator, helping me understand supervised learning through weight updates and error correction. The third activity introduced a multi-layer perceptron with sigmoid activation, extending the model beyond linear separability and showing why hidden layers are necessary for more complex problems such as XOR. The evidence files document these three stages clearly.

This sequence of activities was especially valuable because it made neural network learning more concrete. Rather than treating neural networks as black boxes, I was able to see how outputs are produced, how error is reduced, and how network complexity changes the kinds of problems that can be solved.

Unit 8: Gradient Descent and Optimisation

In Unit 8, I focused on gradient descent as a core optimisation technique used in machine learning. This activity demonstrated how model parameters are updated iteratively in order to reduce the cost function and improve performance.

The notebook shows repeated iterations in which the values of m, b, and the cost function are updated over time, with the cost steadily decreasing as the model improves. It also highlights the relationship between learning rate, number of iterations, and convergence, including the important warning that very large learning rates can miss the global minimum. This made the optimisation process much more intuitive and helped connect mathematical theory to actual model behaviour.

This activity was particularly useful because it made clear that training performance depends not only on the model structure, but also on how the optimisation process is configured. That insight becomes increasingly important in neural networks and deep learning.

Evidence:
View Unit 8 Gradient Descent Evidence

Unit 9: Convolutional Neural Networks (CNNs)

Unit 9 introduced convolutional neural networks through an object recognition activity using the CIFAR-10 dataset. This was a major step forward from structured tabular data into image-based deep learning, where the model learns visual features directly from data.

The notebook evidence shows the loading of CIFAR-10, the shape of the dataset, image visualisation, label inspection, normalisation of image values, categorical encoding, and the creation of validation and training sets. This demonstrates the full preprocessing workflow needed before model training and helps explain why deep learning pipelines require careful preparation rather than simply fitting a model.

This activity strengthened my understanding of how CNNs differ from earlier models. Instead of relying on manually engineered features, CNNs learn spatial representations automatically, which makes them especially powerful for computer vision tasks. It also prepared me well for the more advanced individual deep learning work developed later in the module.

Unit 11: Model Performance Evaluation

This activity focused on evaluating machine learning models using confusion matrices and performance metrics. It reinforced an essential lesson from the module: model quality cannot be judged through accuracy alone.

The uploaded evidence includes confusion matrix outputs as well as measures such as F1-score, accuracy, precision, and recall. Seeing these metrics together made it easier to understand how different evaluation measures highlight different strengths and weaknesses of a model. For example, a model may appear acceptable on accuracy while performing poorly on recall or precision in more critical contexts.

This activity improved my ability to interpret evaluation outputs critically and helped me appreciate why model assessment must always be matched to the context of use. In high-stakes applications, the choice of metric is itself an important professional decision.

Reflection on Activity-Based Learning

The practical activities across this module were essential in helping me move from theoretical understanding to applied machine learning practice. They allowed me to work directly with data, implement models, interpret outputs, and recognise that successful machine learning depends on much more than simply running code.

One of the most important lessons I developed through these activities was the importance of data understanding. In the early activities, exploratory data analysis and regression showed me that model behaviour is shaped by the quality, distribution, and relevance of the data being used. Later activities in clustering, neural networks, and deep learning reinforced that preprocessing, parameter choices, and evaluation methods all significantly influence the reliability of results.

These activities also changed how I think about machine learning professionally. Rather than seeing models only as technical tools, I began to understand them as systems that must be interpreted, evaluated, and applied responsibly. This was particularly meaningful in relation to my interest in healthcare and high-risk environments, where performance, robustness, and safe decision-support are more important than achieving impressive outputs in isolation.

Overall, the activity-based structure of the module helped me build confidence step by step. It strengthened both my technical ability and my critical judgement, which will be important in future academic work and in any real-world application of machine learning.
