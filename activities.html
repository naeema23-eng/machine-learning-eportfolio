Overview

This section presents the practical machine learning activities completed throughout the module. These activities demonstrate my progression from foundational data analysis and statistical modelling to more advanced machine learning techniques, including clustering, neural networks, and deep learning. Beyond implementation, each activity contributed to developing a deeper understanding of model behaviour, limitations, and real-world applicability.

Unit 2 & 3: Exploratory Data Analysis and Regression

This activity focused on understanding data structure, relationships between variables, and the application of regression models to identify patterns.

View Full Activity

I conducted exploratory data analysis (EDA) on the Auto MPG dataset, beginning with data cleaning and handling missing values. Statistical measures such as skewness and kurtosis were analysed to understand data distribution, which is critical before applying any machine learning model.

A key outcome of this analysis was identifying a strong negative correlation between vehicle weight and fuel efficiency (MPG). This relationship was visualised using scatter plots and correlation heatmaps, helping to translate statistical findings into interpretable insights.

A simple linear regression model was then implemented to quantify this relationship. While the model captured the overall trend, this activity highlighted an important limitation: relying on a single predictor reduces model accuracy. This reinforced the importance of feature selection and the need for more complex models in real-world scenarios.

Unit 6: Clustering and Unsupervised Learning

This activity introduced unsupervised learning and the concept of discovering patterns without labelled data.

View Full Activity

Using the Iris dataset, I applied K-Means clustering to group data points based on similarity. Prior to clustering, data standardisation was performed to ensure that features contributed equally to the model.

The elbow method was used to determine the optimal number of clusters, demonstrating how model selection in unsupervised learning relies on interpretation rather than predefined labels. The silhouette score was also used to evaluate cluster quality, providing insight into how well-separated the clusters were.

This activity highlighted that clustering is not purely algorithmic but requires analytical judgment. It also demonstrated how unsupervised learning can be applied in real-world contexts such as customer segmentation, pattern recognition, and anomaly detection.

Unit 7: Neural Networks – Perceptrons

This activity explored the foundations of neural networks through perceptron models.

View Full Activity

I implemented a simple perceptron model to understand how inputs, weights, and activation functions interact to produce outputs. This provided a fundamental understanding of how neural networks process information.

The perceptron was then applied to model an AND logic gate, demonstrating supervised learning through weight adjustments based on prediction errors. This illustrated how models learn iteratively from data.

Finally, a multi-layer perceptron (MLP) was implemented to solve non-linear problems such as XOR. This activity was particularly important because it showed the limitations of single-layer models and the necessity of deeper architectures for complex pattern recognition.

Overall, this unit marked a transition from traditional machine learning approaches to neural network-based learning, forming the foundation for deep learning concepts explored later in the module.

Unit 8: Gradient Descent and Optimisation

Understanding how machine learning models optimise performance using gradient descent.

View Full Activity

This activity focused on gradient descent as a core optimisation algorithm used in machine learning. By iteratively adjusting model parameters, the algorithm reduces the cost function to improve model accuracy.

Through experimentation, I observed how different learning rates affect convergence. A learning rate that is too high leads to instability, while a rate that is too low slows convergence. This demonstrated the importance of hyperparameter tuning in model performance.

The decreasing cost over iterations visually reinforced how optimisation works in practice, bridging theoretical concepts with implementation.

Unit 9: Deep Learning – Convolutional Neural Networks

Application of deep learning techniques to image classification tasks.

View Full Activity

A convolutional neural network (CNN) was implemented using the CIFAR-10 dataset. This marked a significant progression from structured data analysis to unstructured data processing.

CNNs automatically extract spatial features from images through convolutional layers, enabling the model to learn complex patterns such as shapes and textures. This contrasts with earlier models that required manual feature selection.

This activity demonstrated the power of deep learning in real-world applications such as computer vision, autonomous systems, and medical imaging.

Unit 11: Model Performance Evaluation

Evaluating machine learning models using performance metrics.

View Full Activity

This activity focused on evaluating classification models using confusion matrices and derived metrics such as accuracy, precision, recall, and F1-score.

While accuracy provides a general measure, this activity highlighted that it is not always sufficient, especially in imbalanced datasets. Precision and recall provide deeper insight into model performance, particularly in critical applications such as healthcare or fraud detection.

This reinforced the importance of selecting appropriate evaluation metrics depending on the context and the potential impact of model errors.
