Unit 6: Development Team Project – Airbnb Business Analysis

The Unit 6 project focused on analysing the Airbnb NYC 2019 dataset using classical machine learning methods within a team environment. The project addressed a real-world business problem by combining regression and clustering to support pricing analysis and market segmentation. This was my main collaborative project in the module and marked an important transition from smaller technical activities to a more integrated analytical task.

Business Question

The team agreed on the following business question: How can Airbnb optimise pricing strategies in NYC by predicting listing prices using property features and identifying distinct neighbourhood market segments? This question was strategically relevant because pricing strongly influences occupancy, host revenue, and market competitiveness in a platform environment.

My Role and Contribution

In this project, I took the role of Project Lead and Technical Lead. I was responsible for data preprocessing, modelling, clustering, visualisations, and final report integration. This meant that I was not only involved in implementing the technical workflow, but also in ensuring that the final report remained coherent, analytical, and aligned with the marking criteria.

The meeting minutes and team contract confirm this role allocation clearly. They show that I led modelling, clustering, visualisation planning, and final integration, while also coordinating deadlines and maintaining accountability across the team. This role required both technical expertise and collaborative leadership.

Methodology and Analysis

The analysis followed a structured workflow. Irrelevant variables such as IDs and names were removed, missing values were handled, categorical variables were one-hot encoded, and numerical variables were standardised where appropriate. The data was split into training and testing subsets using a fixed random state to preserve reproducibility.

For regression, the project compared Linear Regression, Ridge Regression, and Random Forest. The notebook evidence shows that baseline linear models performed weakly on raw price, while log transformation improved stability. The final report highlights that the Random Forest Regressor achieved the strongest predictive performance at around R² ≈ 0.61, suggesting that Airbnb pricing reflects complex non-linear interactions rather than simple linear relationships.

For clustering, K-Means was used to identify market segments. The Elbow Method indicated that K = 4 offered a good balance between interpretability and segmentation detail. This allowed the team to profile distinct listing groups and connect model outputs to business implications rather than treating clustering as an abstract algorithmic exercise.

Key Learning and Significance

This project strengthened my understanding of classical machine learning workflows, especially the importance of feature engineering, preprocessing, and careful interpretation of results. It also showed that model performance depends heavily on data quality and feature representation. Another key learning point was that teamwork in technical projects requires more than dividing tasks: it also requires consistency, coordination, and active integration to produce a strong final result.

Unit 11: Individual Project – Neural Network Models for Object Recognition Using CIFAR-10

In Unit 11, I independently developed and evaluated deep learning models for object recognition using the CIFAR-10 dataset. This project focused on comparing a custom Convolutional Neural Network (CNN) with a transfer learning model based on MobileNetV2. It required a more advanced level of experimentation than earlier work because it involved model design, validation strategy, data augmentation, and critical interpretation of results.

Project Aim

The aim of the project was not only to compare which model performed better, but to understand why those performance differences emerged. This made the project analytical rather than purely descriptive, with attention to how architecture, dataset constraints, and training strategy interact.

My Individual Contribution

This was a fully independent project, and I was responsible for the entire machine learning pipeline. My work included dataset loading and splitting, preprocessing through normalisation and one-hot encoding, implementation of data augmentation, design of the custom CNN architecture, implementation of transfer learning using MobileNetV2, model training, validation monitoring, optimisation, confusion matrix analysis, classification reporting, and preparation of the final presentation.

I also prepared the presentation slides and explanatory material, which required me to communicate not only what the models achieved, but also the reasons behind model behaviour, limitations, and practical significance.

Methodology and Results

The project used CIFAR-10, consisting of 60,000 colour images across 10 classes. The dataset was split into 40,000 training images, 10,000 validation images, and 10,000 test images. Images were normalised to the 0–1 range and labels were one-hot encoded. Data augmentation was applied through rotation, horizontal flipping, and width/height shifts to improve generalisation.

The custom CNN architecture included stacked convolutional layers, batch normalisation, max pooling, dropout, and dense layers. Training was controlled through early stopping and learning-rate reduction. The uploaded notebook shows that the custom CNN achieved a test accuracy of about 0.8032, with the best validation performance reaching around 80%.

The transfer learning model used MobileNetV2 with resized 96×96 inputs and a frozen pretrained base. According to my project notes and presentation materials, this approach performed slightly better, reaching roughly 81–82% accuracy. However, the performance gain was modest rather than dramatic.

Critical Interpretation

A key insight from this project was that increasing model complexity does not automatically produce a proportionally better result. Although MobileNetV2 slightly outperformed the custom CNN, the improvement was limited. This suggested that the main limitation was not architecture alone, but the characteristics of the dataset itself, especially the low 32×32 image resolution and the visual similarity between several classes.

The confusion matrix and classification outputs revealed consistent misclassification between visually similar classes, especially cat vs dog and bird vs frog. This showed that deep learning performance is strongly constrained by data quality and feature separability. Data augmentation improved robustness, but it did not resolve deeper semantic ambiguity between classes.

Another important lesson was the role of validation. Validation curves made it possible to identify where performance began to plateau, which helped prevent overfitting and reinforced the importance of separating training, validation, and testing for trustworthy evaluation.
